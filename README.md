# LuminaCore

LuminaCore เป็นปลั๊กอินสำหรับเซิร์ฟเวอร์ Luminaris ที่รวบรวมระบบอำนวยความสะดวกและตรวจส่องการทำงานภายในเซิร์ฟเวอร์เข้าด้วยกันภายใต้สถาปัตยกรรมแบบโมดูลาร์ (Modular Architecture)

## สถาปัตยกรรมโมดูลาร์ (Modular Architecture)

ทุกระบบการทำงานภายใน LuminaCore ถูกแยกออกจากกันเป็นอิสระ โดยแบ่งออกเป็น 3 หมวดหมู่ย่อย ได้แก่ System (ระบบสอดส่องหลังบ้าน), Activate (ระบบที่ทำงานผ่านคีย์ลัดย่อตัว), และ Features (ระบบอำนวยความสะดวกทางกายภาพ) แต่ละระบบจะมีไฟล์กำหนดค่า (YML) ของตัวเอง เพื่อให้แอดมินสามารถเปิด/ปิดการทำงาน, เปิด/ปิดระบบสิทธิ์ (Permission), เลือกรูปแบบการส่งข้อความแจ้งเตือน (Chat หรือ Actionbar), ปรับแต่งเสียงพร้อมค่า Pitch, รวมถึงระดับความละเอียดของหลอดพลังความคืบหน้า (Progress Bar) ได้อย่างอิสระ พร้อมรองรับการเปิดปิดโหมดผ่านหน้าต่าง GUI ของแอดมินโดยตรง

## รายการฟีเจอร์ของระบบย่อยทั้งหมด

### หมวดหมู่ System (ระบบตรวจสอบพื้นหลังและส่งข้อมูลบันทึกผ่าน Discord Webhook)

#### 1. BlockNotifier (ระบบตรวจสอบการขุดแร่สำคัญ)

- การทำงาน: เมื่อผู้เล่นขุดแร่สำคัญที่กำหนดไว้ในคอนฟิก (เช่น แร่เพชร หรือแร่เนเธอไรต์) ระบบจะทำการส่งรายละเอียด (ชื่อผู้เล่น พิกัด โลก ชนิดบล็อก และจำนวนแร่ในกลุ่มที่ขุดได้ในจุดนั้น) ไปยัง Discord Webhook
- การตั้งค่า: กำหนดรายชื่อบล็อกแร่ที่ตรวจจับ และกำหนดที่อยู่ URL Discord Webhook

#### 2. CreativeMonitor (ระบบควบคุมพฤติกรรมโหมดสร้างสรรค์)

- การทำงาน: ตรวจสอบพฤติกรรมของผู้เล่นเมื่อเปิดใช้งานโหมดสร้างสรรค์ (Creative Mode) โดยจะคอยสอดส่องการหยิบเสกไอเทมจากรายการบล็อกและการทิ้งไอเทมลงพื้น แล้วทำการส่งข้อมูลแจ้งเตือนไปยัง Discord Webhook
- การตั้งค่า: เปิด/ปิดการตรวจจับการหยิบและโยนแยกจากกันได้อิสระ

#### 3. RconMonitor (ระบบเฝ้าระวัง Rcon)

- การทำงาน: ดักจับและตรวจเช็คเหตุการณ์บันทึกการพยายามเชื่อมต่อเข้าระบบ Rcon รวมถึงเมื่อมีการรันคำสั่งใดๆ ผ่าน Rcon สำเร็จ แล้วแจ้งเตือนความเคลื่อนไหวไปยัง Discord Webhook ทันที
- การตั้งค่า: เลือกดักฟังคำสั่งหรือความพยายามเข้าใช้งาน Rcon แยกจากกัน

#### 4. UnknownCommand (ระบบดักจับคำสั่งที่ไม่มีในระบบ)

- การทำงาน: เมื่อผู้เล่นพิมพ์คำสั่งที่ไม่มีอยู่จริงบนเซิร์ฟเวอร์ หรือคำสั่งอันตรายเฝ้าระวังพิเศษ (เช่น /op, /gamemode) ระบบจะส่งข้อมูลคำสั่งนั้นและพิกัดผู้เล่นไปเก็บที่ Discord Webhook เพื่อสอดส่องพฤติกรรมแปลกปลอม
- การตั้งค่า: ระบุรายชื่อคำสั่งเฝ้าระวังพิเศษเฉพาะเจาะจงได้ในคอนฟิก

#### 5. AntiOp (ระบบรักษาความปลอดภัยสิทธิ์ผู้ดูแลระบบ)

- การทำงาน: ระบบรักษาความปลอดภัยขั้นสูงสำหรับเฝ้าระวังและป้องกันการสวมรอยสิทธิ์ผู้ดูแลระบบ (OP) หรือการมอบสิทธิ์ระดับสูงที่ไม่ถูกต้อง
  - การตรวจสอบ Whitelist: บัญชีที่ได้รับสิทธิ์ OP หรือสิทธิ์ควบคุมสูงสุด `*` แต่ไม่ได้อยู่ในรายชื่อ Whitelist จะถูกระงับสิทธิ์และระงับการเข้าใช้งานทันทีตามมาตรการความปลอดภัย
  - การยืนยันสิทธิ์ในเกม: ผู้เล่นที่ได้รับอนุญาตเมื่อเข้าสู่ระบบด้วยสิทธิ์ OP จะต้องทำการกรอกรหัสผ่านยืนยันสิทธิ์ผ่านหน้าต่างยืนยันตัวตน (Native Dialog UI) ภายในระยะเวลาที่กำหนด โดยทิศทางการมองจะถูกจำกัดชั่วคราวและคำสั่งต่าง ๆ จะถูกปิดกั้นอย่างเงียบจนกว่าจะยืนยันตัวตนสำเร็จ
  - ป้องกันการมอบสิทธิ์โดยมิชอบ (Privilege Escalation): ตรวจจับและสกัดกั้นการรันคำสั่งมอบ OP หรือคำสั่งในสิทธิ์ LuckPerms ระดับสูงที่รันผ่าน Console หรือผู้เล่นที่ไม่ได้รับอนุญาต และดำเนินการระงับบัญชีคู่กรณีและผู้รับสิทธิ์ทันที
- การตั้งค่า: ตั้งค่ารหัสผ่านยืนยันตัวตน, Whitelist ผู้มีสิทธิ์, LuckPerms บล็อกกรุ๊ป, ระยะเวลายืนยันตัวตน, โหมดเกมหลังยืนยัน, และข้อความแสดง Log ทาง Console

### หมวดหมู่ Activate (ระบบสลับเปิดปิดและสั่งงานผ่านคีย์ลัดย่อตัว)

ระบบกลุ่มนี้สามารถทำงานได้ผ่านคีย์ลัดย่อตัวสะสม ซึ่งรองรับผู้เล่นจาก Bedrock (Geyser) และ Java ที่สลับคีย์ลัดปุ่มย่อเป็น ctrl หรือปุ่มอื่นได้ โดยผู้เล่นจะเห็นหลอดพลังความคืบหน้า (Progress Bar) บน Actionbar และเล่นเสียงความถี่ Pitch สูงขึ้นตามการกดย่อตัวสะสม

#### 6. GlowingTwerk (ระบบย่อตัวเร่งการเติบโต)

- ปุ่มควบคุม: ย่อตัวซ้ำๆ ใกล้พืชเพาะปลูก
- การทำงาน: ทำการเร่งการเจริญเติบโตของพืชรอบข้างในรัศมีแบบสุ่ม (เทียบเท่าการใช้ Bone Meal) พร้อมแสดงพาร์ทิเคิลสีเขียวอ่อนรอบตัวบล็อกที่โตสำเร็จ
- การตั้งค่า: กำหนดโอกาสในการเติบโต ระยะเวลาคูลดาวน์ และชนิดพืชที่รองรับ

#### 7. TreeCapitator (ระบบตัดไม้หมดต้น)

- ปุ่มควบคุม: ถือขวานกดย่อตัวสะสม 10 ครั้ง (ตั้งค่าจำนวนครั้งได้)
- การทำงาน: สลับเปิด/ปิดโหมดตัดไม้ เมื่อเปิดโหมดแล้วตัดบล็อกไม้ ท่อนไม้ตระกูลเดียวกันที่เชื่อมต่ออยู่รอบข้างและเหนือบล็อกขึ้นไปจะโดนทำลายลงมาทั้งหมดทันที (จำกัดสูงสุด 128 บล็อก)
- การตั้งค่า: ปรับรัศมีการสแกน การลบใบไม้โดยรอบทันที และการลงต้นอ่อนชนิดเดิมกลับดินล่างสุดอัตโนมัติ

#### 8. AutoPlanter (ระบบปลูกพืชรอบตัว)

- ปุ่มควบคุม: ถือเมล็ดพืชหรือผลผลิตเกษตรสำหรับปลูก กดย่อตัวสะสม 10 ครั้ง
- การทำงาน: สแกนหาดินเพาะปลูก Farmland ว่างในรัศมีรอบตัว แล้วปลูกเมล็ดพืชจากในมือผู้เล่นลงไปจนกว่าเมล็ดพืชจะหมดหรือดินถูกปลูกเต็ม
- การตั้งค่า: กำหนดช่วงรัศมีสแกนรอบตัว

#### 9. SpawnChecker (ระบบตรวจสอบจุดสปอน Monster)

- ปุ่มควบคุม: ถือคบเพลิง (Torch) กดย่อตัวสะสม 10 ครั้ง
- การทำงาน: ตรวจสอบจุดอับแสง (ระดับแสงสว่างประดิษฐ์เท่ากับ 0) ในรัศมี 8 บล็อกรอบตัว ซึ่งเข้าเงื่อนไขการเกิดมอนสเตอร์ศัตรู แล้วทำการสปอนอนุภาคเปลวไฟชี้จุดเหล่านั้นเป็นเวลา 5 วินาที
- การตั้งค่า: ระยะรัศมีการแผ่แสงที่ตรวจสอบ

#### 10. TimeWeatherViewer (ระบบแสดงเวลาและสภาพอากาศ)

- ปุ่มควบคุม: ถือนาฬิกา (Clock) กดย่อตัวสะสม 10 ครั้ง
- การทำงาน: แปลง ticks ปัจจุบันของโลกออกมาแสดงผลในรูปแบบชั่วโมงและนาที (HH:mm) พร้อมระบุสภาพภูมิอากาศ (ท้องฟ้าโปร่ง, ฝนตก, พายุฝน)
- การตั้งค่า: รูปแบบข้อความแสดงเวลา

#### 11. CoordDirectionViewer (ระบบแสดงพิกัดและทิศทาง)

- ปุ่มควบคุม: ถือเข็มทิศ (Compass) กดย่อตัวสะสม 10 ครั้ง
- การทำงาน: รายงานค่าพิกัดปัจจุบัน X, Y, Z ของตัวละครผู้เล่น พร้อมระบุทิศทางหลักที่กำลังหันไป (North, East, South, West) อ้างอิงตามองศา Yaw

#### 12. NetherCalculator (ระบบคำนวณพิกัดสลับมิติ)

- ปุ่มควบคุม: ถือออบซิเดียน (Obsidian) กดย่อตัวสะสม 10 ครั้ง
- การทำงาน: ตรวจจับมิติปัจจุบันของผู้เล่น หากอยู่ใน Overworld จะคำนวณหาตำแหน่งสร้างประตูพิกัดโลกนรก Nether (หาร X, Z ด้วย 8) และหากอยู่ในโลกนรก Nether จะคูณพิกัด Overworld (คูณ X, Z ด้วย 8)

#### 13. DistanceMeasurer (ระบบวัดระยะทาง)

- ปุ่มควบคุม: ถือเข็มทิศ (Compass) กดย่อตัวแล้วคลิกขวา
- การทำงาน: เปิดโหมดการวัดระยะทาง (Measure Mode) จากนั้นให้ผู้เล่นคลิกขวาบล็อกจุดที่หนึ่งและคลิกขวาบล็อกจุดที่สอง ระบบจะรายงานระยะทางตรง Euclidean และระยะตามเส้นทางเดิน Manhattan ให้ทราบทันที

#### 14. VeinMiner (ระบบขุดแร่ยกกลุ่ม)

- ปุ่มควบคุม: ถือที่ขุดแร่ (Pickaxe) กดย่อตัวสะสม 10 ครั้ง
- การทำงาน: สลับเปิด/ปิดโหมดทำเหมือง เมื่อขุดบล็อกแร่ชนิดใดๆ กลุ่มก้อนแร่ประเภทเดียวกันในรัศมีข้างเคียงจะโดนทำลายลงมาทั้งหมดพร้อมกันสูงสุด 64 บล็อก โดยสูญเสียความทนทานเครื่องมือและรับผลของเสน่ห์มนต์สะกดตามจริง

### หมวดหมู่ Features (ระบบอำนวยความสะดวกเสริมและการแปลงสภาพทางกายภาพ)

ระบบกลุ่มนี้ทำงานแบบตอบสนองอัตโนมัติเมื่อเงื่อนไขและเหตุการณ์เฉพาะในเกมเข้าข่าย

#### 15. ColorSign (เขียนป้ายอักษรสี)

- การทำงาน: แปลงรหัสสีแบบดั้งเดิม (&a, &l), รหัส Hex (&#FF5555) และข้อความสไตล์ MiniMessage (<gradient:red:blue>) บนป้ายชื่อทันทีที่เขียนหรือบันทึกข้อมูลเสร็จสิ้น
- การตั้งค่า: เลือกเปิด/ปิดการอนุญาตประเภทของรหัสสี (Hex, Legacy, MiniMessage) และเล่นเสียงตอบรับ

#### 16. ConcreteConverter (แปลงผงคอนกรีตเป็นคอนกรีตในน้ำ)

- การทำงาน: เมื่อโยนไอเทมผงคอนกรีต (Concrete Powder) จมลงในน้ำเป็นเวลา 10 วินาที ไอเทมจะแปลงสภาพกลายเป็นบล็อกคอนกรีตแข็งสีเดียวกันทันที
- การตั้งค่า: ระยะเวลาแปลงสภาพ (วินาที), บังคับต้องเป็นบล็อกน้ำนิ่งเท่านั้นหรือไม่, การแสดงความคืบหน้าหลอดเวลาถอยหลัง (Progress Bar) บน ActionBar ของผู้เล่นรอบตัว, และเล่นเสียง Pitch ไล่เสียงขึ้นตามวินาทีการสุกของบล็อก

#### 17. AnimalAutoHeal (ฟื้นฟูเลือดสัตว์เลี้ยงและสัตว์ฟาร์ม)

- การทำงาน: หากสัตว์เลี้ยงหรือสัตว์ในฟาร์มที่รองรับอยู่ในระยะตรวจสอบ ได้รับความเสียหายแต่ไม่ได้เข้าสู่การต่อสู้ล่าสุดใน 10 วินาที ระบบจะฟื้นฟูพลังชีวิตให้ 1 HP ทุกๆ 2 วินาทีอัตโนมัติพร้อมเอฟเฟกต์รูปหัวใจ
- การตั้งค่า: คูลดาวน์ดาเมจ, อัตราการฟื้นฟู, รัศมีการทำงาน, และระดับเสียงที่ความถี่ Pitch จะยิ่งแหลมขึ้นตามอัตราร้อยละพลังชีวิตที่เต็มขึ้นเรื่อยๆ ของสัตว์ตัวนั้น

#### 18. AutoReplenish (เติมบล็อกและอุปกรณ์ในมืออัตโนมัติ)

- การทำงาน: เติมบล็อกที่วางจนหมด หรือขว้างปาหิมะ/ไข่/ขวด EXP/ยาปา, ให้อาหารสัตว์, ตักน้ำ/ลาวา/เทนมจากถัง, หรือเมื่ออุปกรณ์เครื่องมือพังคามือ โดยการสแกนหาไอเทมหรือเครื่องมือชนิดเดียวกันในกระเป๋าเก็บของมาใส่ทดแทนในช่องมือที่ใช้งานให้อัตโนมัติทันที
- การตั้งค่า: เปิด/ปิดการตรวจสอบแยกตามประเภท เช่น บล็อก, ไอเทมปา, ถังน้ำ, เครื่องมือพัง

#### 19. AutoEatReplenish (เติมอาหารอัตโนมัติ)

- การทำงาน: เมื่อผู้เล่นกินอาหารคำสุดท้ายในมือหลักจนหมดลง ระบบจะตรวจหาและเติมอาหารชนิดเดียวกันมาใส่ในมือ หากหมดจะหยิบอาหารประเภทอื่นในกระเป๋ามาสลับใส่ทดแทนตามลำดับความต้องการ โดยจะป้องกันการหยิบอาหารอันตรายที่เป็นพิษเด็ดขาด
- การตั้งค่า: ลำดับประเภทอาหารที่ลำเลียงมาใส่ก่อนหลัง (Priority List), สวิตช์อนุญาตอาหารไม่ปลอดภัย, และรายชื่อบล็อกแบล็กลิสต์อาหารดิบ/มีพิษ

#### 20. ConfirmDrop (ยืนยันสิทธิ์ป้องกันการโยนอุปกรณ์ทิ้ง)

- การทำงาน: ป้องกันการพลาดกดปุ่มคีย์ทิ้งไอเทมสำคัญ (เช่น อุปกรณ์เครื่องมือเนเธอไรต์/ไดอา, ชุดเกราะหลัก, และปีก Elytra) โดยยกเลิกการทิ้งในครั้งแรก พร้อมแสดงหลอดความคืบหน้า (Progress Bar) แบบเคลื่อนไหวถอยหลังบน ActionBar และผู้เล่นต้องกดปุ่มทิ้งซ้ำภายใน 3 วินาทีเพื่อทิ้งจริง
- การตั้งค่า: ระยะเวลารอกดซ้ำ, การจำกัดเฉพาะของที่มีมนต์สะกด (Enchanted), รายการวัสดุคุ้มครอง, และสไตล์เสียงแจ้งเตือนครั้งแรก/ทิ้งสำเร็จ

#### 21. AnimalOwner (ตรวจสอบสิทธิ์เจ้าของสัตว์เลี้ยง)

- การทำงาน: กดย่อตัวคลิกขวา (Shift + Right Click) ที่สัตว์เชื่องได้ เพื่อขอตรวจสอบรายชื่อผู้เล่นที่เป็นเจ้าของสัตว์ตัวดังกล่าว
- การตั้งค่า: ประเภทสัตว์ที่ตรวจสอบได้, รูปแบบข้อความแสดงชื่อเจ้าของ, ปริมาณ HP และสูงสุดของสัตว์เลี้ยงตัวนั้น

#### 22. NetherSponge (ฟองน้ำแห้งในนรก)

- การทำงาน: เมื่อทำการโยนไอเทมฟองน้ำเปียก (Wet Sponge) ทิ้งลงในมิติ Nether ตัวไอเทมจะดูดความร้อนระเหยน้ำกลายเป็นฟองน้ำธรรมดาทันที พร้อมเล่นเสียงน้ำเดือดแห้งและปล่อยเอฟเฟกต์ควัน
- การตั้งค่า: ชนิดพาร์ทิเคิลควัน, ปริมาณ และเสียงแจ้งผล

#### 23. PathConverter (แปลงบล็อกทางดินกลับเป็นบล็อกดิน)

- การทำงาน: ใช้พลั่ว (Shovel) คลิกขวาที่บล็อกทางเดินดิน (Dirt Path) บล็อกจะคืนสภาพกลับเป็นบล็อกดินปกติ (Dirt) โดยจะลดความทนทานพลั่วลง 1 หน่วย (คิดคำนวณตามระดับ Unbreaking บนพลั่วด้วย)
- การตั้งค่า: บล็อกเป้าหมาย (เช่น DIRT หรือ COARSE_DIRT), ค่าสูญเสียความทนทาน, และเอฟเฟกต์ดินฝุ่นแตกกระจาย

#### 24. FarmlandConverter (แปลงดินเกษตรกลับเป็นบล็อกดิน)

- การทำงาน: ใช้จอบ (Hoe) คลิกขวาที่บล็อกดินแปลงเกษตร (Farmland) บล็อกจะคืนสภาพกลับเป็นบล็อกดินปกติ (Dirt) โดยจะลดความทนทานจอบลง 1 หน่วย (คิดคำนวณตามระดับ Unbreaking บนจอบด้วย)
- การตั้งค่า: บล็อกเป้าหมาย, ค่าสูญเสียความทนทาน, พาร์ทิเคิลและเสียง

## วิธีการติดตั้งและการใช้งาน

1. นำไฟล์ `LuminaCore.jar` ที่บิลด์สำเร็จไปวางลงในไดเรกทอรี `plugins/` ของเซิร์ฟเวอร์
2. สตาร์ทเซิร์ฟเวอร์ ตัวปลั๊กอินจะทำการโหลดตัวเองและแตกไฟล์การตั้งค่าเริ่มต้นของโมดูลย่อยและ `config.yml` ลงในโฟลเดอร์ `plugins/LuminaCore/`
3. ผู้เล่นที่เป็นแอดมิน (มีสิทธิ์ `luminacore.admin` หรือ OP) สามารถพิมพ์คำสั่ง `/luminacore` ในเกมเพื่อเปิดหน้าต่าง GUI เพื่อเลือกรัน คลิกเปิด/ปิด สถานะเปิดใช้งานของโมดูลย่อยทั้งหมดได้ทันที
4. พิมพ์คำสั่ง `/luminacore reload` เพื่อบังคับรีโหลดไฟล์ตั้งค่าของทุกโมดูลโดยไม่ต้องรีสตาร์ทเซิร์ฟเวอร์

## วิธีการบิลด์ปลั๊กอินเอง (Building the Plugin)

คุณสามารถนำซอร์สโค้ดนี้ไปบิลด์ได้ด้วย Gradle โดยแยกออกเป็น 2 รูปแบบตามต้องการ:

### 1. การบิลด์แบบเต็ม (Full Build)

เป็นการบิลด์ปลั๊กอิน `LuminaCore` รวมทุกโมดูลเข้าด้วยกันเป็นไฟล์เดียว เพื่อใช้เป็นตัวหลักของเซิร์ฟเวอร์

- รันคำสั่ง:
  ```bash
  gradle jar
  ```
- ผลลัพธ์: ไฟล์ปลั๊กอินหลักจะอยู่ที่ `build/libs/LuminaCore.jar` (หรือมีเวอร์ชันต่อท้าย)

### 2. การบิลด์แบบเดี่ยว (Standalone Build)

เป็นการแยกบิลด์แต่ละโมดูลย่อยออกมาเป็นปลั๊กอินเดี่ยว ๆ อิสระจากกัน (เช่น นำไปใช้เฉพาะตัวขุดแร่ ตัดไม้ หรือระบบความปลอดภัย โดยไม่ต้องใช้โมดูลอื่น)

- **บิลด์ทุกโมดูลย่อยพร้อมกัน**:
  ```bash
  gradle jarAllStandalone
  ```
- **บิลด์เฉพาะโมดูลที่ต้องการ**:
  ```bash
  gradle jar[ชื่อโมดูล]
  ```
  _(ตัวอย่างเช่น `gradle jarAntiOp` หรือ `gradle jarTreeCapitator`)_
- ผลลัพธ์: ไฟล์ปลั๊กอินย่อยจะอยู่ที่ `build/libs/` โดยมีชื่อ Classifier ลงท้ายด้วย `-standalone-[ชื่อโมดูล]`

## วิธีการข้ามการตรวจสอบลิขสิทธิ์ (Bypass License Verification)

หากต้องการเปิดใช้งานปลั๊กอินโดยไม่ต้องกรอกหรือเชื่อมต่อตรวจสอบคีย์ลิขสิทธิ์ (License Verification) ให้ปฏิบัติดังนี้ก่อนทำการบิลด์ปลั๊กอิน:

1. เปิดไฟล์ LicenseManager.kt
2. แก้ไขค่าตัวแปร `BYPASS` ให้เป็น `true`:
   ```kotlin
   private const val BYPASS = true
   ```
3. ดำเนินการบิลด์ปลั๊กอินตามปกติ ระบบการตรวจสอบคีย์จะถูกข้ามโดยสมบูรณ์และแสดงข้อความแจ้งผลบน Console ว่า
   `License Key Bypass is ENABLED. Skipping license verification.`
